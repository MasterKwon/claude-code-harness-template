롤백 맵 — 실패해도 돌아갈 수 있다

문제 발견 위치에 따라 어느 단계로 돌아갈지가 파이프라인에 명시되어 있습니다.

문제 발견 위치원인조치
테스트 실패 API 구현 오류 /refine-build-api → 리뷰 → 테스트 재실행
UAT FAIL 기능 누락 /refine-design-* → 구현 → 테스트 → 배포
운영 배포 실패 환경 설정 오류 직접 수정 → /deploy-prd 재실행

롤백의 핵심 원칙

  • 문제가 생긴 Phase의 직전 단계로만 돌아갑니다
  • 더 앞 단계를 건드리면 연쇄 변경이 발생하므로 금지
  • 롤백 경로가 명시되어 있어 패닉 없이 대응 가능
"어디로 돌아가야 할지 모르는 상황"이 가장 위험합니다.
롤백 맵은 그 불안을 제거하는 안전망입니다.

/grill-me 단계 인지형 자유 호출 v2.4.16+

/grill-me는 분석 단계 시작 전뿐 아니라 설계·구현·테스트 진행 중에도 자유롭게 호출할 수 있습니다.

호출 시점 자동 인식

grill-me 가 Step 0 에서 현재 단계를 추정하고 사용자에게 1회 확인:

  • analyze 단계 → docs/00.input/grill-result.md
  • design 단계 → docs/02.design/grill-decisions.md
  • build 단계 → docs/03.build/grill-decisions.md
  • test 단계 → docs/05.test/grill-decisions.md

후속 스킬 자동 반영

design-* / build-* / test-* 스킬은 사전 동작에서 해당 단계의 grill-decisions.md를 읽어 결정 사항을 자동 반영합니다. 사용자가 별도로 전달하지 않아도 됩니다.

같은 단계에서 여러 번 호출하면 append 누적 (덮어쓰기 X)

사용 예시 (설계 단계)

/design-api 진행 중 페이지네이션 방식이 모호 → /grill-me 호출

하네스 설계 원칙

이 원칙들이 지켜질 때 하네스가 오래 동작합니다.

✂️

Surgical Changes — 수술적 변경

요청받은 범위 밖의 코드는 건드리지 않습니다. 변경은 정교하고 최소한이어야 합니다.

🔧

Harness Engineering — 환경 개선

실수가 발생하면 사과 대신 같은 실수가 반복되지 않도록 환경을 즉시 수정합니다. CLAUDE.md·스킬 파일 업데이트.

🎯

Simplicity First — 단순함 우선

불필요한 추상화와 과한 설계를 지양합니다. 현재 문제를 해결하는 가장 단순한 코드를 선택합니다.

🧠

Think Before Coding — 먼저 계획

코드를 작성하기 전 가설과 실행 계획을 수립하고 확인받습니다.
