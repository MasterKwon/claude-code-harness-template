스킬은 어떻게 태어나는가

①

"이 작업을 매번 반복하고 있다"

코드 리뷰할 때마다 같은 체크리스트 / 배포할 때마다 같은 절차 확인 / 회의 후 항상 같은 양식으로 정리

②

"이걸 명확하게 말로 표현할 수 있다"

어떤 입력을 받아서 / 무엇을 확인하고 / 어떤 결과를 만들어야 하는지

③

"AI에게 만들어달라고 한다"

파일 구조를 몰라도 됩니다. 필요를 말할 수 있으면 충분합니다.

스킬 파일을 직접 쓸 필요 없습니다.
무엇이 필요한지 판단하는 것이 전부입니다.

워크샵 — 내 업무에서 반복 찾기

지금 하는 업무에서 매번 같은 방식으로 반복하는 작업을 하나 골라보세요. (3분)

힌트 — 개발자라면

  • PR 올리기 전 셀프 체크리스트
  • API 명세 작성 (매번 같은 양식)
  • 에러 로그 분석 패턴
  • 신규 기능의 테스트 케이스 작성
  • 코드 리뷰 코멘트 패턴

힌트 — 기획자라면

  • 화면 기획서 작성 양식
  • 요구사항 → 개발팀 전달 정리
  • QA 체크리스트 작성
  • 고객 피드백 분류 및 정리
  • 스프린트 회의록 양식
