어제는 잘 됐는데 오늘은 왜 다른 결과가 나올까?

같은 AI인데 매번 다른 품질.

"프롬프트를 잘 써야 한다"

프롬프트 엔지니어링을 배워야 하나?

진짜 원인은?

내가 무엇을 원하는지 명확하지 않은 것이 문제입니다.

ChatGPT vs Cursor vs Claude Code

항목 ChatGPT Cursor Claude Code
실행 환경 웹 브라우저 IDE (VS Code 기반) 터미널 (CLI)
파일 접근 ❌ 없음 ✅ 프로젝트 전체 @codebase ✅ 프로젝트 전체
환경 제어 ❌ 없음 ✅ .cursor/rules/*.mdc
파일 타입별 규칙 자동 첨부		 ✅ CLAUDE.md
프로젝트 전역 지속 컨텍스트
재사용 지시문 ❌ 없음 ⚠️ Rules (체인 불가) ✅ 스킬 + 파이프라인 체인
에이전트 실행 ❌ 없음 ✅ IDE 내 Agent 모드
터미널·파일 멀티스텝 실행		 ✅ 터미널 독립 실행
CI/CD·헤드리스 실행 가능
파이프라인 체인 ❌ ❌ 단계 자동 연쇄 없음 ✅ 분석→설계→구현→배포 자동
Cursor는 IDE 안에서 개발자를 보조하는 도구입니다.
Claude Code는 개발 워크플로 전체를 자동화하는 에이전트입니다.

오늘 쓸 용어 — 딱 3개

스킬

AI에게 주는 역할 카드. "요구사항 분석 담당자처럼 행동해라" 같은 지시문 묶음.

파이프라인

분석 → 설계 → 구현 → 테스트 → 배포를 순서대로 자동 실행하는 흐름.

하네스

AI가 일관된 품질로 실수 없이 일하도록 씌우는 안전 장치 전체. 스킬 + 파이프라인 + 품질 게이트.

실습 중 모델이 자동 전환됩니다 → Opus 4.7 — 인터뷰 (최고 품질) Sonnet 4.6 — 분석·설계 (균형)
오늘 실습에서 이 세 가지가 눈앞에서 동작하는 것을 직접 확인합니다.

오늘 체험할 파이프라인

실습 주제: CMS 컴포넌트 빌더 — 텍스트·이미지·버튼·배너 4종 컴포넌트를 페이지에 추가/편집/삭제하는 기능

환경 준비 — 3분

아래 명령어를 터미널에 복사-붙여넣기 하세요.

Step 1 — 템플릿 클론

git clone https://github.com/MasterKwon/claude-code-harness-template.git my-workshop
cd my-workshop

Step 2 — Claude Code 실행

claude

Step 3 — 실습 디렉터리 확인

ls docs/

✅ docs/00.input/ 폴더가 보이면 준비 완료입니다.
✅ Claude Code 터미널에 > 프롬프트가 나타나면 시작 가능합니다.

이 하나의 저장소가 오늘 실습의 모든 도구와 스킬을 포함하고 있습니다.
실습 1 /grill-me Opus 4.7 ⏱ 8분

AI가 한 번에 하나씩 질문을 던집니다. 자신의 이해대로 솔직하게 답하세요.

시작 멘트 — 실행 후 아래 내용을 붙여넣으세요

CMS 페이지 편집기에서 사용할 컴포넌트 빌더를 개발하려고 합니다.
관리자가 텍스트, 이미지, 버튼, 배너 4가지 컴포넌트를
페이지에 자유롭게 배치하고 속성을 설정할 수 있어야 합니다.
저장과 미리보기 기능도 필요합니다.
AI가 물을 수 있는 질문 예시
  • 속성 편집은 누가 할 수 있습니까? 권장: 관리자만
  • 컴포넌트 수 제한이 있습니까? 권장: 없음
  • 비로그인 미리보기가 가능해야 합니까? 권장: 가능
출력
  • docs/00.input/grill-result.md
  • 합의된 결정 사항 목록
  • 보류된 결정 목록
질문에 깊게 답할수록, 이후 설계 결과물이 달라집니다.
실습 2 /analyze-requirements Sonnet 4.6 ⏱ 4분

인터뷰 결과를 읽고 구조화된 요구사항 문서를 자동으로 만듭니다.

입력 (자동 참조)
  • docs/00.input/grill-result.md
  • 방금 완성한 인터뷰 결과
출력
  • docs/01.analyze/requirements.md
  • 기능 목록, 화면 목록, 비기능 요구사항
  • 용어 정의

실행 방법

/analyze-requirements

실행 후 docs/01.analyze/requirements.md 파일을 열어 결과를 확인하세요.
기능이 몇 개로 분류됐는지, 화면이 몇 개로 나왔는지 옆 사람과 비교해보세요.

실습 3 /design-db Sonnet 4.6 ⏱ 4분

요구사항을 읽고 DB 테이블 구조를 자동으로 설계합니다.

입력 (자동 참조)
  • docs/01.analyze/requirements.md
  • 방금 생성된 요구사항 문서
출력
  • docs/02.design/db.md
  • 테이블 정의, 컬럼, 관계(ERD)
  • 인덱스 설계

주목할 포인트

  • 컴포넌트 타입을 어떻게 저장했나? (ENUM vs 별도 테이블)
  • 페이지와 컴포넌트의 관계가 어떻게 표현됐나? (1:N vs N:M)
  • 순서(order) 컬럼이 있는가?

grill-me에서 다르게 답한 사람은 이 테이블 구조도 다르게 나왔을 수 있습니다.

실습 4 /design-prompt-gen Sonnet 4.6 ⏱ 3분

요구사항의 각 화면을 분석해서 Claude Design에 붙여넣을 프롬프트를 자동 생성합니다.

입력 (자동 참조)
  • docs/01.analyze/requirements.md
출력
  • docs/02.design/design-prompts.md
  • 화면별 디자인 프롬프트
  • 검토 체크리스트

생성되는 프롬프트 예시

컴포넌트 편집 화면을 만들어줘.

목적: 관리자가 페이지의 컴포넌트(텍스트·이미지·버튼·배너)를
      추가하고 속성을 설정한다.

주요 요소:
- 왼쪽 패널: 컴포넌트 타입 선택 목록
- 중앙: 캔버스 (컴포넌트 배치 미리보기)
- 오른쪽 패널: 선택된 컴포넌트 속성 편집 폼

디자인 시스템: shadcn/ui, Tailwind CSS
실습 5 claude.ai/design ⏱ 3분

생성된 프롬프트를 Claude Design에 붙여넣으면 시각적 화면 목업이 즉시 만들어집니다.

Step 1

docs/02.design/design-prompts.md 파일을 열고 원하는 화면의 프롬프트를 복사합니다.

Step 2

브라우저에서 claude.ai/design 접속 후 프롬프트를 붙여넣습니다.

Step 3

생성된 화면 목업을 확인하고, 옆 사람 결과와 비교합니다.

같은 주제로 시작했지만, grill-me에서 어떻게 답했느냐에 따라 전혀 다른 화면이 나옵니다.

결과 비교 — 옆 사람과 견줘보세요

요구사항

  • 기능이 몇 개로 분류됐나?
  • 화면이 몇 개로 나왔나?
  • 비기능 요구사항이 있나?

DB 설계

  • 테이블이 몇 개인가?
  • 컴포넌트 타입을 어떻게 저장했나?
  • 순서(order) 컬럼이 있나?

화면 목업

  • 레이아웃 구성이 다른가?
  • 포함된 요소가 다른가?
  • 어느 쪽이 더 완성도가 높나?

같은 주제, 다른 결과

AI가 다른 게 아닙니다.
grill-me에서 내가 어떻게 답했느냐의 차이입니다.
