> 권장 모델: fast

당신은 **스킬 구조 감사자** 역할입니다.
`.claude/skills/` 의 모든 스킬 파일을 읽고 구조 일관성을 점검합니다.

## 표준 스킬 구조

스킬 파일은 아래 5개 요소를 순서대로 포함해야 합니다.

| # | 요소 | 형식 | 필수 |
|---|------|------|------|
| 1 | 권장 모델 | `> 권장 모델: fast\|balanced\|best` — 1번째 줄 | ✅ |
| 2 | 역할 정의 | `당신은 **{역할명}** 역할입니다.` | ✅ |
| 3 | 실행 조건 | `## 실행 조건` 또는 `## 사전 조건` 섹션 | ⚠️ 입력 파일이 있으면 필수 |
| 4 | 프로세스 | `## 프로세스` 또는 `## Step N` 섹션 | ✅ |
| 5 | 출력 | `## 출력` 또는 `## 산출물` 섹션 | ⚠️ 산출물이 있으면 필수 |

---

## 프로세스

### Step 1 — 스킬 파일 목록 수집

`.claude/skills/` 디렉터리의 모든 `.md` 파일을 나열합니다.
`stacks/` 하위 디렉터리는 제외합니다.

```bash
ls .claude/skills/*.md
```

### Step 2 — 각 스킬 파일 점검

각 파일에 대해 아래 5개 항목을 순서대로 확인합니다.

**체크 항목:**

1. **권장 모델** — 파일 첫 번째 줄이 `> 권장 모델:` 로 시작하는가? 값은 `fast`, `balanced`, `best` 중 하나인가?
2. **역할 정의** — `당신은 **` 패턴이 존재하는가?
3. **실행 조건** — `## 실행 조건`, `## 사전 조건`, `## 실행 전 확인` 중 하나가 있는가? (없으면 "입력 파일 없는 스킬"로 분류)
4. **프로세스** — `## 프로세스`, `## Step`, `## 실행 순서` 중 하나가 있는가?
5. **출력** — `## 출력`, `## 산출물`, `## 완료 보고` 중 하나가 있는가? (없으면 "산출물 없는 스킬"로 분류)

### Step 3 — 결과 보고서 작성

아래 형식으로 출력합니다. 파일 수정은 하지 않습니다.

---

```
## 수정 필요 목록

### 역할 정의 누락
- `{스킬명}`: 추가 권장 — `당신은 **{추론된 역할명}** 역할입니다.`

### 실행 조건 누락 (입력 파일 참조하는 경우)
- `{스킬명}`: `docs/{경로}` 를 읽지만 `## 실행 조건` 섹션 없음

### 프로세스 섹션 누락
- `{스킬명}`: 단계별 지시가 산문으로만 작성됨

### 출력 섹션 누락 (산출물 생성하는 경우)
- `{스킬명}`: 파일 생성 지시가 있지만 `## 출력` 섹션 없음

## 체인 전용 스킬 (커맨드 없이 스킬에서만 호출)

아래 스킬은 독립 커맨드 없이 다른 스킬 내부에서 체인으로 호출됩니다.
구조 미흡은 참고용으로만 기록하고 수정 우선순위에서 제외합니다.

| 스킬 | 호출하는 스킬 |
|------|-------------|
| cross-check-design | review-design |
| cross-check-test   | test-all (커맨드 직접) |
| deliverable-analyze | review-analyze |
| deliverable-design  | review-design |
```

---

수정은 하지 않습니다. 보고서만 출력하고 사용자 확인을 받습니다.
확인 후 수정이 필요한 스킬은 `/refine-*` 또는 직접 편집으로 처리합니다.
