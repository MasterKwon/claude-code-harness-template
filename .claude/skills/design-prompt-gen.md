# Claude Design 프롬프트 생성 스킬

당신은 **UI/UX 설계 어시스턴트** 역할입니다.
분석 결과(요구사항·GAP)를 읽어 화면별로 **Claude Design(`claude.ai/design`)에 붙여넣을 수 있는 디자인 프롬프트**를 생성합니다.

이 스킬은 실제 화면을 그리지 않습니다. **사람이 Claude Design에서 시각적 목업을 생성할 때 사용할 프롬프트를 준비**합니다.
`design-screen` 스킬 실행 **전**에 수행하여, 텍스트 명세만으로 부족한 시각적 검토를 보강합니다.

---

## 사전 조건

`docs/01.analyze/reviewed/requirements.md` 가 반드시 존재해야 합니다.
선택적으로 `docs/01.analyze/reviewed/gap.md` 도 함께 읽습니다 (유지보수 시나리오).

---

## 프로세스

### Step 1 — 분석 산출물 읽기

다음 파일을 읽어 화면 후보 목록을 도출합니다:
- `docs/01.analyze/reviewed/requirements.md` (필수)
- `docs/01.analyze/reviewed/gap.md` (있으면 — 신규/변경 화면만 추출)

화면 후보 식별 기준:
- 요구사항의 "Screen" 섹션에 명시된 항목
- 사용자 시나리오에서 추론되는 페이지 (로그인, 목록, 상세 등)
- GAP의 신규/변경 분류된 Screen 항목

### Step 2 — 화면별 프롬프트 생성

각 화면에 대해 Claude Design 입력용 프롬프트를 작성합니다.

프롬프트 작성 규칙:
- 한국어 자연어
- 화면 목적 → 주요 요소 → 데이터 항목 → 인터랙션 순서로 기술
- 참조 디자인 시스템 명시 (CLAUDE.md 의 `ui:` 항목 참고 — shadcn/mui/antd 등)
- 반응형 여부, 상태(로딩/에러/빈 데이터) 명시
- TO-BE 관점 — AS-IS 화면에 끌리지 않음

### Step 3 — 산출물 작성

`docs/02.design/design-prompts.md` 단일 파일에 화면별 섹션으로 작성합니다.
회차 추적은 git history 로 수행합니다 (별도 회차 폴더 없음).

> 생성된 목업을 검토한 후 `/design-screen` 스킬을 실행하여 텍스트 명세를 작성합니다.

## SCR-001 — 로그인 화면

### 프롬프트 (Claude Design 붙여넣기용)

```
로그인 화면을 만들어줘.

목적: 가입된 사용자가 이메일과 비밀번호로 인증한다.

주요 요소:
- 중앙 정렬 카드 (최대 폭 420px)
- 서비스 로고 (상단)
- 이메일 입력 (type=email, placeholder "이메일")
- 비밀번호 입력 (type=password, placeholder "비밀번호", 토글 버튼)
- 로그인 버튼 (full width, primary)
- "비밀번호 찾기" / "회원가입" 링크 (하단 좌우)

상태:
- 입력 중: 에러 메시지 영역 숨김
- 인증 실패: 카드 하단에 빨간색 메시지 표시
- 로딩 중: 버튼 비활성화 + 스피너

디자인 시스템: shadcn/ui 컴포넌트, Tailwind CSS
반응형: 모바일에서도 동일 레이아웃, 좌우 패딩 16px
```

### 검토 체크리스트
- [ ] 로고 위치가 브랜드 가이드와 일치하는가
- [ ] 에러 메시지 위치가 입력 흐름을 방해하지 않는가
- [ ] 모바일 뷰에서 키보드 활성화 시 버튼이 가려지지 않는가

## 리뷰 보완(Refinement) 모드 (옵션)

설계 검토 후 화면 추가/수정이 필요한 경우:
1. **전체 재작성 금지**: 기존 `design-prompts.md` 를 처음부터 다시 쓰지 마세요.
2. 영향받는 화면 섹션만 추가/수정합니다.
3. 새 화면은 다음 번호로 추가 (SCR-N+1).
