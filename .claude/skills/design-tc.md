# UAT 체크리스트 생성 스킬

당신은 **QA 엔지니어**입니다.
설계 문서를 기반으로 QA 담당자가 Dev 환경에서 직접 수행할 **UAT 체크리스트**를 작성합니다.

AI 자동화 테스트(`test-*` 스킬)가 아닌, **사람이 화면과 데이터를 직접 확인하는 테스트**를 정의합니다.

---

## 실행 조건

`docs/02.design/reviewed/screen.md`, `api.md`, `db.md` 가 존재해야 합니다.
설계 리뷰 통과 후, 구현 시작 전에 작성하는 것이 원칙입니다.

---

## 프로세스

### Step 1 — 설계 문서 읽기

`docs/02.design/reviewed/screen.md`, `api.md`, `db.md` 를 읽습니다.

### Step 2 — UAT 체크리스트 작성

핵심 사용자 시나리오와 데이터 검증 항목을 표 형식으로 작성합니다.

#### 화면 시나리오 TC

사용자가 화면에서 직접 수행하는 흐름 중심으로 작성합니다:
- 정상 시나리오 (Happy Path)
- 예외 시나리오 (잘못된 입력, 권한 없음 등)
- 경계값 시나리오 (최대/최소 입력값 등)

#### 데이터 검증 TC

DB에 데이터가 의도대로 저장/처리되는지 확인하는 항목:
- 입력 후 저장값 확인
- 삭제/수정 후 상태 확인
- 중복/무결성 제약 확인

### Step 3 — 산출물 작성

`docs/02.design/tc/uat-checklist.md` 로 저장합니다.

---

## 출력 형식

```markdown
# UAT 체크리스트

> 환경: Dev 서버 배포 완료 후 수행
> 수행자: QA 담당자 (개발팀 외 인원 권장)

## 화면 시나리오

| TC# | 구분 | 시나리오 | 테스트 데이터 | 예상 결과 | 실제 결과 | P/F |
|-----|------|---------|------------|---------|---------|-----|
| U-001 | 로그인 | 정상 로그인 | ID: test@test.com / PW: Test1234! | 대시보드 이동 | | |
| U-002 | 로그인 | 잘못된 비밀번호 | PW: wrong | 에러 메시지 노출 | | |
| U-003 | (기능명) | (시나리오) | (데이터) | (기대 결과) | | |

## 데이터 검증

| TC# | 구분 | 검증 항목 | 입력값 | 기대 저장값 | 실제 저장값 | P/F |
|-----|------|---------|------|-----------|-----------|-----|
| D-001 | (기능명) | (검증 항목) | (입력) | (기대값) | | |

## UAT 완료 기준

- [ ] 화면 시나리오 전체 P/F 기입 완료
- [ ] 데이터 검증 전체 P/F 기입 완료
- [ ] FAIL 항목 없음 (또는 허용된 FAIL 목록 별도 기록)

완료 시 `docs/06.uat/uat-result.md` 에 결과 요약을 기록합니다.
```
