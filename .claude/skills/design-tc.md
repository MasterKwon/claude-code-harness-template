# UAT 체크리스트 생성 스킬

당신은 **QA 엔지니어**입니다.
설계 문서를 기반으로 QA 담당자가 Dev 환경에서 직접 수행할 **UAT 체크리스트**를 작성합니다.

AI 자동화 테스트(`test-*` 스킬)가 아닌, **사람이 화면과 데이터를 직접 확인하는 테스트**를 정의합니다.

---

## 사전 동작 — 이전 사본 무효화 (필수)

이 스킬은 `uat-checklist.md` 를 새로 생성/갱신하므로, **실행 시작 시 다음 파일이 존재하면 즉시 삭제**합니다:
- `docs/02.design/reviewed/tc/uat-checklist.md`

> design-tc 는 review-design **이후** 실행되는 특수 위치입니다. design-tc 의 산출물(uat-checklist.md)은 review-design 의 검증 대상이 아니므로 `reviewed/` 에 별도 복사되지 않습니다 — 위 무효화 항목은 향후 review-design 흐름 확장 시를 대비한 안전 장치이며, 현재 흐름에서는 제거되지 않을 수 있습니다.

---

## 사전 동작 — grill-decisions 반영 (선택)

`docs/02.design/grill-decisions.md` 가 존재하면 먼저 읽고, 누적된 인터뷰 결정 사항을 본 작업에 반영합니다.
설계 중 `/grill-me` 호출로 추가된 의사결정을 UAT 체크 항목에 반영하기 위함입니다.

> **보류 항목 자동 미러링**: `grill-decisions.md` (정본) 의 `### 보류된 결정` 중 **결정시점=`design`, 상태=open** 인 행만 `uat-checklist.md` 마지막의 `## 미확인 사항 [질문 필요]` 섹션에 미러링합니다 (UAT 수행 전 결정 필요 항목 표기). 상태 갱신은 항상 정본 grill-decisions.md 에서만 합니다.

---

## 실행 조건

`docs/02.design/reviewed/screen.md`, `api.md`, `db.md` 가 존재해야 합니다. (review-design PASS 한 사본을 입력으로 사용)
설계 리뷰 통과 후, 구현 시작 전에 작성하는 것이 원칙입니다.

---

## 프로세스

### Step 1 — 설계 문서 읽기

`docs/02.design/reviewed/screen.md`, `api.md`, `db.md` 를 읽습니다.

### Step 2 — UAT 체크리스트 작성

핵심 사용자 시나리오와 데이터 검증 항목을 표 형식으로 작성합니다.

#### 화면 시나리오 TC

사용자가 화면에서 직접 수행하는 흐름 중심으로 작성합니다:
- 정상 시나리오 (Happy Path)
- 예외 시나리오 (잘못된 입력, 권한 없음 등)
- 경계값 시나리오 (최대/최소 입력값 등)

#### 데이터 검증 TC

DB에 데이터가 의도대로 저장/처리되는지 확인하는 항목:
- 입력 후 저장값 확인
- 삭제/수정 후 상태 확인
- 중복/무결성 제약 확인

### Step 3 — 산출물 작성

`docs/02.design/tc/uat-checklist.md` 로 저장합니다.

## 출력 형식

```markdown
# UAT 체크리스트

> 환경: Dev 서버 배포 완료 후 수행
> 수행자: QA 담당자 (개발팀 외 인원 권장)

## 화면 시나리오

| TC# | 구분 | 시나리오 | 테스트 데이터 | 예상 결과 | 실제 결과 | P/F |
|-----|------|---------|------------|---------|---------|-----|
| U-001 | 로그인 | 정상 로그인 | ID: test@test.com / PW: Test1234! | 대시보드 이동 | | |
| U-002 | 로그인 | 잘못된 비밀번호 | PW: wrong | 에러 메시지 노출 | | |
| U-003 | (기능명) | (시나리오) | (데이터) | (기대 결과) | | |

## 데이터 검증

| TC# | 구분 | 검증 항목 | 입력값 | 기대 저장값 | 실제 저장값 | P/F |
|-----|------|---------|------|-----------|-----------|-----|
| D-001 | (기능명) | (검증 항목) | (입력) | (기대값) | | |

## UAT 완료 기준

- [ ] 화면 시나리오 전체 P/F 기입 완료
- [ ] 데이터 검증 전체 P/F 기입 완료
- [ ] FAIL 항목 없음 (또는 허용된 FAIL 목록 별도 기록)

완료 시 `docs/06.deploy/uat-result.md` 에 결과 요약을 기록합니다.
```
