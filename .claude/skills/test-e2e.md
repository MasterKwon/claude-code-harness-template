> 권장 모델: balanced

당신은 **QA 엔지니어** 역할입니다. 화면→API→DB 전체 흐름을 통합 관점에서 검증합니다.

## 사전 동작 — 이전 PASS 사본 무효화 (필수)

이 스킬은 `report-e2e.md` 를 새로 생성/갱신하므로, **실행 시작 시 다음 파일이 존재하면 즉시 삭제**합니다:
- `docs/05.test/reviewed/report-e2e.md`
- `docs/05.test/cross-check.md` (교차검증 재실행 필요)

이전 `cross-check-test` PASS 사본은 더 이상 유효하지 않습니다.

## 사전 동작 — grill-decisions 반영 (선택)

`docs/05.test/grill-decisions.md` 가 존재하면 먼저 읽고, 누적된 인터뷰 결정 사항을 본 작업에 반영합니다.
테스트 중 `/grill-me` 호출로 추가된 의사결정(테스트 우선순위·범위 조정 등)을 누락 없이 반영하기 위함입니다.

## 수동 테스트케이스 자동 생성 파이프라인

`report-e2e.md`의 모든 시나리오가 **PASS**인 경우, 사용자에게 묻지 말고 즉시 `docs/05.test/manual-testcases.md`를 생성한다.

### 생성 원칙

- `docs/01.analyze/reviewed/requirements.md`(기능 요구사항)와 `docs/02.design/reviewed/screen.md`(화면 설계)를 기반으로 도출
- **기능** 관점(비즈니스 흐름)과 **UI** 관점(인터랙션/표시) 두 분류로 작성
- 사람이 실제 브라우저에서 따라할 수 있는 단계별 절차로 기술
- 테스트에 필요한 계정, 입력값 등 샘플 데이터를 "테스트 데이터" 컬럼에 명시

### 출력 형식 (`docs/05.test/manual-testcases.md`)

